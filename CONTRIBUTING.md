# 執筆・提案ガイド

『現代戦術と指揮』への提案、訂正、参考資料の共有を歓迎します。

## Issueで歓迎する内容

- 誤字、リンク切れ、用語の誤り
- 一次資料または信頼できる研究に基づく事実関係の訂正
- 図版の読みやすさ、アクセシビリティに関する改善
- 章構成、演習問題、参考文献の提案
- 複数の解釈がある論点への異論・補足

提案には、可能であれば該当ファイル、見出し、根拠となる資料、提案する修正文を含めてください。

## Pull Requestの方針

1. 一つのPull Requestでは、できるだけ一つの論点を扱う。
2. 事実の追加・変更には出典を付ける。
3. 引用は必要最小限とし、引用箇所と著者自身の説明を区別する。
4. 図版にはタイトル、説明、キャプションを付ける。
5. 実在の作戦に転用可能な詳細な攻撃手順、個別標的情報、武器の製造・改造手順は含めない。
6. 特定の国・組織の宣伝を目的とせず、主張と根拠を分けて記述する。
7. リポジトリ所有者・投稿者・関係者の個人情報を本文や図版へ転載しない。

## レビュー時の確認項目

- **正確性**：出典は主張を実際に支えているか。
- **時点**：制度、役職、ドクトリン、紛争情勢はいつの情報か。
- **均衡**：反対資料や別解釈を不当に省略していないか。
- **明瞭性**：一般読者が定義なしの略語や専門語で迷わないか。
- **図解品質**：色だけに依存せず、文字・線種・形でも区別できるか。
- **安全性**：教育的な分析を越えて、現実の危害を容易にする内容になっていないか。
- **プライバシー**：個人を特定・推測できる情報や文書メタデータが含まれていないか。

## 本文・図版の基本形式

- 本文：Markdown
- 図版：原則として編集可能なSVG。写真などが必要な場合は、権利と出典を確認する。
- ファイル名：半角英数字とハイフンを推奨
- 画像の代替説明：SVGでは title と desc、Markdownでは説明的な代替テキストを付ける
- 参考文献：資料名、発行主体、年版、URLを記載する

## 提案の反映

IssueやPull Requestは、出典、論旨、著作権、安全性、全体構成との整合を確認してから反映します。提案が採用されない場合でも、重要な異論は本文の注記や「別の見方」として紹介することがあります。
